💾 用户数据功能说明

本地存储 + 在线提交 — 完全掌控你的补充数据

📍 功能入口

每个工具页面（故障代码、PLC指令、电气符号）都有一个 「📝 我的补充」 折叠面板，点击展开后可以看到操作按钮。

💾 方式一：保存到本地（推荐日常使用）

原理：数据存储在浏览器的 localStorage 中，不上传到任何服务器。

添加数据：

  1. 点击 + 添加数据 按钮
  2. 填写表单（品牌、代码、名称等）
  3. 点击 💾 保存本地 按钮
  4. 数据会立即出现在列表中，带蓝色「用户补充」标签

删除数据：

  • 每条数据右侧有「删除」按钮，点击即可删除

⚠️ 注意事项：

  • 数据只保存在当前浏览器中
  • 清除浏览器缓存会丢失数据
  • 不同浏览器/设备之间数据不互通
  • 适合个人临时记录，不适合长期存储
📥 方式二：导入导出 JSON（数据备份/迁移）

导出 JSON：

  1. 点击 📥 导出JSON 按钮
  2. 浏览器会下载一个 .json 文件
  3. 文件包含你添加的所有本地数据

导入 JSON：

  1. 点击 📤 导入JSON 按钮
  2. 选择之前导出的 .json 文件
  3. 数据会自动合并到现有列表中

💡 使用场景：

  • 备份：换电脑前先导出，到新电脑再导入
  • 分享：导出 JSON 发给同事，同事导入即可使用
  • 跨浏览器：从 Chrome 导出，导入到 Firefox
  • 批量编辑：导出 → 用文本编辑器修改 → 重新导入
🌐 方式三：提交到服务器（社区共享）

原理：数据上传到 Cloudflare 全球网络，审核后会加入网站公共数据。

提交数据：

  1. 填写表单后，点击 🌐 提交到服务器 按钮
  2. 显示「✅ 已提交，审核后表示」成功
  3. 管理员审核通过后，数据会出现在网站公共列表中

查看在线提交：

  1. 点击 🌐 在线提交 按钮
  2. 查看当前已提交的待审核数据

⚠️ 注意事项：

  • 提交后需要管理员审核才会显示
  • 审核通过的数据会成为网站永久公共数据
  • 请确保数据准确，避免误导他人
  • 无需注册，无需登录，直接提交
❓ 常见问题

Q: 本地数据存储在哪里？

浏览器 localStorage，路径：F12 → Application → Local Storage → 当前网站

Q: 数据会自动同步吗？

不会。本地数据只在本机浏览器，不同步到其他设备。

Q: 清除浏览器缓存会丢失数据吗？

会。建议定期导出 JSON 备份。

Q: 提交到服务器的数据可以删除吗？

已审核入库的数据不能自助删除，需联系管理员。未审核的可以联系管理员删除。

Q: 为什么有些数据有蓝色边框？

蓝色边框 + 「用户补充」标签 = 你手动添加的数据（本地存储）

Q: 导入的 JSON 格式不对怎么办？

确保是数组格式，每项包含必要字段（如故障代码必须有 brand、code、name）。可参考导出的 JSON 文件结构。
